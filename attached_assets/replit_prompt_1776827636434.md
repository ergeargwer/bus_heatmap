# Replit AI 修改指令：新增定位選站 + Autocomplete 搜尋功能

## 背景說明

這是一個 Vite + React + TypeScript 專案，使用 DeckGL 與 MapLibre 顯示台灣公車路線熱圖，資料來源為 TDX API（台灣交通資料平台）。

目前使用者只能在搜尋框輸入站名關鍵字送出後才能查詢路線。  
本次要新增兩個功能：

1. **定位按鈕**：偵測 GPS 位置，找出半徑 100 公尺內的站位，讓使用者選擇後顯示路線
2. **站名 Autocomplete**：在搜尋框輸入文字時，即時顯示下拉候選清單，使用者點選即觸發查詢，不需按 Enter

---

## 修改範圍

### 一、`src/tdxApi.ts` — 新增兩個函式

#### 1. `getNearbyStations(lon, lat, radiusMeters)`

呼叫 TDX API 查詢指定座標半徑 100 公尺內的公車站位清單。

```
GET https://tdx.transportdata.tw/api/basic/v2/Bus/Station/City/Taipei
  ?$spatialFilter=nearby({lat},{lon},{radiusMeters})
  &$select=StationUID,StationName,StationPosition
  &$format=JSON
```

回傳格式：
```ts
export interface NearbyStation {
  uid: string;
  name: string;
  lon: number;
  lat: number;
}
```

回傳陣列 `NearbyStation[]`，若 API 無結果則回傳空陣列。

#### 2. `getStationSuggestions(keyword)`

輸入關鍵字，查詢站名包含該字串的前 10 筆站位（用於 autocomplete）。

```
GET https://tdx.transportdata.tw/api/basic/v2/Bus/Station/City/Taipei
  ?$filter=contains(StationName/Zh_tw,'{keyword}')
  &$select=StationUID,StationName,StationPosition
  &$top=10
  &$format=JSON
```

回傳格式同上 `NearbyStation[]`。

#### 搜尋框改為 Autocomplete

- 使用者輸入文字時，debounce 300ms 後呼叫 `getStationSuggestions(keyword)`，將結果寫入 `suggestions`
- 顯示下拉清單（`showSuggestions === true` 且 `suggestions.length > 0`）
- 使用者點選某筆候選站名時：
  1. 將 `searchInput` 設為該站名
  2. 隱藏下拉清單
  3. 直接呼叫原有的 `handleSearch` 邏輯（以站名查詢路線）
- 使用者在輸入框外點擊時，隱藏下拉清單

下拉清單 DOM 結構範例：

```tsx
{showSuggestions && suggestions.length > 0 && (
  <div className="autocomplete-dropdown">
    {suggestions.map(s => (
      <div
        key={s.uid}
        className="autocomplete-item"
        onMouseDown={() => handleSelectSuggestion(s.name)}
      >
        {s.name}
      </div>
    ))}
  </div>
)}
```

注意：使用 `onMouseDown` 而非 `onClick`，避免 input `onBlur` 比點擊先觸發導致清單消失。

#### 定位按鈕

在搜尋框旁新增一個定位按鈕（使用 `lucide-react` 的 `LocateFixed` icon）。

點擊流程：
1. 設 `isLocating = true`，顯示 loading 動畫
2. 呼叫 `navigator.geolocation.getCurrentPosition()`
3. 取得座標後呼叫 `getNearbyStations(lon, lat, 100)`
4. 若有結果：
   - 將結果寫入 `nearbyStations`
   - 設 `showNearbyList = true`，顯示附近站位選擇清單（覆蓋在 routes-list 上方或取代之）
   - 地圖飛梭至使用者位置
5. 若無結果：`alert('附近 100 公尺內未找到公車站位')`
6. 最終設 `isLocating = false`

附近站位清單 DOM 結構範例：

```tsx
{showNearbyList && (
  <div className="nearby-list">
    <div className="nearby-list-header">
      <span>附近站位（100m）</span>
      <button onClick={() => setShowNearbyList(false)}>x</button>
    </div>
    {nearbyStations.map(s => (
      <div
        key={s.uid}
        className="nearby-item"
        onClick={() => handleSelectNearbyStation(s.name)}
      >
        {s.name}
      </div>
    ))}
  </div>
)}
```

`handleSelectNearbyStation(name)` 的行為：
1. 設 `searchInput = name`
2. 設 `showNearbyList = false`
3. 呼叫原有 `handleSearch` 邏輯（以站名查詢路線並飛梭）

## 注意事項

- 所有程式碼禁止使用 emoji
- 維持現有 TDX token 快取機制（`accessToken` 變數），新函式同樣呼叫 `getTDXToken()`
- Autocomplete debounce 必須清除上一個 timer，避免競態條件
- 定位功能若瀏覽器不支援 `navigator.geolocation`，顯示 `alert('您的瀏覽器不支援定位功能')`
- 不更動現有的路線切換 toggle、動畫邏輯、LiveBus 輪詢邏輯
